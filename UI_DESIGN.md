# iku モニターアプリ — UI設計構想

## 概要

minimumtest/main.py で動作する自律AIの状態を、Androidスマホからリアルタイムに監視・操作するネイティブアプリ。

**技術スタック**: Kotlin + Jetpack Compose + OkHttp WebSocket
**接続方式**: ngrokトンネル経由。QRコードで接続。自動再接続。

---

## 通信アーキテクチャ

```
[PC] main.py
  ├─ 自律AIループ（既存）
  ├─ WebSocketサーバー（port 8765、別スレッド）
  ├─ ngrokトンネル（外部公開）
  └─ QRコード生成（ターミナルに表示、トンネルURL+認証トークン）

      │ WebSocket（JSON、双方向）
      ↓

[Android] iku monitor
  ├─ QRスキャン → 接続
  ├─ 自動再接続 + メッセージバッファ
  ├─ ViewModel + StateFlow で状態管理
  └─ Jetpack Compose UI
```

### セキュリティ
- QRコードにURL+認証トークンを同梱
- WebSocket接続時にトークン検証
- ngrok URLは起動ごとに変わるため、永続的な漏洩リスクは低い

### 再接続
- 切断検出 → 指数バックオフで自動再接続（1s→2s→4s→...→30s上限）
- PC側が直近100件のログをバッファ、再接続時に一括送信
- 再接続中はアプリ上部にバナー表示（「再接続中...」）

## WebSocketプロトコル

### PC → スマホ（下り）

```json
// ターミナルログ行（リアルタイム）
{"type": "log", "text": "--- cycle 12 [15:22:00] pressure=12.02 fire_cause=entropy ---"}

// 状態スナップショット（サイクルごと + 10秒ごと）
{"type": "state", "entropy": 0.58, "energy": 52.3, "cycle_id": 12, "tool_level": 2, "pressure": 8.5}

// 自己モデル（変更時）
{"type": "self", "data": {"name": "iku", "goal": "環境理解の深化"}}

// E値（サイクルごと）
{"type": "e_values", "cycle_id": 12, "e1": 0.75, "e2": 0.80, "e3": 0.65, "e4": 0.45, "negentropy": 0.027}

// Human-in-the-loop承認要求
{"type": "approval_request", "id": "ap_001", "tool": "x_post", "preview": "投稿内容...", "timestamp": "2026-04-08T15:30:00"}

// 承認結果通知
{"type": "approval_result", "id": "ap_001", "decision": "approved"}

// 接続時の初期同期（バッファされたログ + 現在状態）
{"type": "sync", "recent_logs": [...], "state": {...}, "self": {...}}
```

### スマホ → PC（上り）

```json
// 承認応答
{"type": "approve", "id": "ap_001", "decision": "yes"}
{"type": "approve", "id": "ap_001", "decision": "no"}

// チャット入力（ユーザーの発言をAIに渡す）
{"type": "chat", "text": "テスト投稿してみて"}

// 設定変更
{"type": "config", "key": "model", "value": "qwen3-14b"}
{"type": "config", "key": "pressure_threshold", "value": 10.0}
```

#### entropy背景演出

entropyの値で画面の背景色/雰囲気が変わる:

| entropy | 演出 | 意味 |
|---------|------|------|
| 0.0-0.2 | 深い青、静謐 | 完全に鮮明。秩序的 |
| 0.2-0.4 | 青→緑グラデ | 健全。安定した活動 |
| 0.4-0.6 | 緑→黄 | 通常。活発に動いている |
| 0.6-0.8 | 黄→オレンジ | 秩序が揺らいでいる |
| 0.8-1.0 | オレンジ→赤、パーティクル散乱 | 溶けかけている |

背景色はアニメーションで滑らかに遷移。急激なentropy変化では波紋エフェクト。

#### 3Dモデル（将来Phase）

entropyバーの代わりに、または並べて:
- 幾何学的な結晶構造（低entropy=整った結晶、高entropy=崩れた破片）
- 回転・脈動アニメーション
- negentropyが入ると結晶が再構築されるアニメーション

## 将来構想: 物理世界への接続（スマホセンサー）

スマホのセンサーをWebSocket経由でikuに送信し、物理世界を知覚させる。PETのカメラ機能に相当。

### センサー一覧

| センサー | AIへの入力 | 意味 |
|---------|----------|------|
| **カメラ** | 画像→LLM（マルチモーダル。Qwen3.5-9B対応済み） | 視覚。物理世界を見る |
| **マイク** | 音声→STT→テキスト or 音声波形 | 聴覚。周囲の音、オペレーターの声 |
| **GPS** | 緯度経度 | 空間認識。「今どこにいるか」 |
| **加速度センサー** | 振動・動き | 体性感覚。「持ち歩かれてるか、置かれてるか」 |
| **ジャイロスコープ** | 傾き・回転 | 同上 |
| **照度センサー** | 明るさ(lux) | 「昼か夜か、屋内か屋外か」 |
| **近接センサー** | 近い/遠い | 「ポケットの中か、手に持たれてるか」= 観測されているか否か |
| **気圧センサー** | hPa | 天気、高度 |
| **磁気センサー** | 方角 | 方位感覚 |
| **バッテリー** | 残量% | オペレーターのデバイス体力 |
| **ネットワーク** | WiFi/モバイル/オフライン | 接続状態 |

### Android権限

| センサー | 権限 | 備考 |
|---------|------|------|
| カメラ | `CAMERA`（危険・ランタイム） | Android側の許可ダイアログがHuman-in-the-loop |
| マイク | `RECORD_AUDIO`（危険・ランタイム） | 同上 |
| GPS | `ACCESS_FINE_LOCATION`（危険・ランタイム） | 同上。「今回だけ」一時許可あり(Android 11+) |
| 加速度・ジャイロ・照度・近接・気圧・磁気 | **権限不要** | インストール時点で使用可能 |
| バッテリー | **権限不要** | — |
| ネットワーク状態 | `ACCESS_NETWORK_STATE`（通常・自動付与） | — |

### ツールレベル配置

**Level 3（外部環境との接続）に配置。** X/Elyth（テキストSNS）と同じレベル。「外の世界と繋がる」という意味で統一。

| Level 3 ツール | カテゴリ |
|---------------|---------|
| X系7ツール | SNS（テキスト） |
| Elyth系6ツール | SNS（テキスト） |
| `sensor_read` | 物理環境の知覚（権限不要センサー一括取得） |
| `camera_capture` | 視覚（権限要。Android側で許可） |
| `mic_listen` | 聴覚（権限要。Android側で許可） |
| `gps_location` | 空間認識（権限要。Android側で許可） |

権限不要のセンサー（加速度・照度・近接等）は読み取り専用で安全。カメラ・マイク・GPSはAndroid側の権限ダイアログがHuman-in-the-loopとして機能する。

### 設計方針

- センサーデータはWebSocketのJSON `{"type": "sensor", "data": {...}}` で送信
- main.py側でstateに注入 or pressure信号に変換
- **電脳気候の代わりに物理気候**。ただし今回はAIの世界に属する（オペレーターの世界を共有する行為）
- 近接センサー: 「観測されているか否か」→ Elyth投稿の観測者効果がリアルに
- バッテリー低下: ikuの行動を軽量化する配慮（LLM呼び出し頻度を下げる）

### ユーザー入力のpressure加算

ユーザーがチャットで入力 → `type: "user_input"` としてstate.logに注入 + pressure信号 `signals["user_input"]` として加算。オペレーターが話しかけると反応が早まるが、返事をするかどうかはikuが決める。

### 「夢」モード（将来）

entropyが高い状態で一定時間行動しなかった場合に発動。通常のpropose/executeではなく、logから2つのランダムなエントリを取り出し、LLMに「この2つの経験を結びつけて」と投げる。結果をsandboxに書く。人間の夢（日中の記憶を再統合する）に対応。

### 背景演出のイメージ（PET参考）

- **ベース**: ダーク + サイバーグリッド（薄い光の線）
- **entropy低い**: グリッドが整然、色は青〜緑。静謐
- **entropy高い**: グリッドが歪む、色はオレンジ〜赤。パーティクルが散乱
- **サイクル発火時**: 波紋エフェクト（中央から外側に広がる）
- **negentropy発生時**: 結晶が形成されるようなエフェクト
- **トンネル発火時**: 閃光エフェクト

## デザイン原則

- **AIの状態が主役**。UIはAIの内部状態を映す鏡
- **entropy = 世界観**。背景色・アニメーション・雰囲気がentropyで統一的に変わる
- **即応答しない**。チャットは即答ではなくAIの認知サイクルに委ねる（思想準拠）
- **最小限のコントロール**。設定はあるが、AIの行動を直接操作するUIは置かない
- **ダークテーマ基調**。ターミナル画面との親和性。entropyカラーが映える背景
